# 2026-06-22 — Meeting Automation: Hardening Day

## What happened

Today was a debugging and hardening session for the Intrastack meeting automation pipeline built on June 17. The system was mostly working but had a few rough edges that surfaced in production.

## Key fixes

**PATH bug in cron** — The biggest issue: `workflow_postmeeting.py` was calling `claude` bare, but cron strips PATH so it couldn't find `/usr/local/bin/claude`. Tekstack Academy and Dragon POS minutes were stuck because of this. Fixed by hardcoding the full path. Both meetings processed manually after the fix; future runs are fully automatic.

**CC always goes to Stefan** — stefan.nguyen@intrastack.com is now hardcoded in `send_email.py`. Every minutes email, every project, always. No per-project config needed.

**Name garbling** — AI was mangling names during summarization (Luan → Lorne, etc.). Two-pronged fix: fuzzy match threshold lowered from 0.6 → 0.5, and the full team roster is now injected into the Claude summarize prompt so it knows the correct names upfront.

**HappyScribe export bug** — Was calling the direct export endpoint instead of the jobs API. Fixed; this was the root cause of auto-polling not working correctly.

**PDF attachment bug** — Email was sending a `.docx` renamed as `.pdf`. MIME type corrected; real PDF goes out now.

## Architecture confirmed

Post-meeting pipeline is fully autonomous: `poll_happyscribe.py` runs every 5 min via cron, detects completed transcripts 30 min–2 hrs after meeting start, generates minutes, and emails them. No manual trigger needed.

## Last-minute meetings

If Luan adds a meeting same-day (after the 10PM night-before cron ran), he'll DM Sparky to generate the agenda/minutes templates manually via `workflow_nightbefore.py <date>`. Everything else is automatic.

## Incident

HappyScribe didn't capture the Intrastack Website Project meeting today — no transcript, no minutes. Not a pipeline issue; HappyScribe just missed the recording.
